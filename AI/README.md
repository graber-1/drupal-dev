# AI Documentation Folder

This folder contains documentation and resources specifically for AI assistants working on this project.

## Purpose

The AI folder serves as a centralized location for:
- Memory banks and context documentation
- AI-specific instructions and guidelines
- Project knowledge bases
- Development patterns and conventions

## Contents

### MEMORY_BANK.md
**Primary documentation for AI assistants**

Contains comprehensive project information including:
- Project overview and technology stack
- Custom module development guidelines
- LMS entity types and architecture
- Development workflows and best practices
- Code quality standards
- Common tasks and troubleshooting

**When to reference:** At the start of any new task to understand project context

**When to update:** When significant architectural changes occur, new patterns emerge, or important discoveries are made

## Usage Guidelines

### For AI Assistants
1. **Start of new task:** Read `MEMORY_BANK.md` to understand project context
2. **During development:** Reference relevant sections for patterns and conventions
3. **After major changes:** Update the memory bank with new information
4. **Before completion:** Ensure any new patterns or learnings are documented

### For Developers
- This folder is safe to commit to version control
- Use it to communicate project context to AI assistants
- Update memory banks when project structure or conventions change
- Add new documentation files as needed

## File Naming Conventions

- Use descriptive names with underscores or hyphens
- Use `.md` extension for markdown documentation
- Prefix with purpose when relevant (e.g., `MEMORY_BANK.md`, `PATTERNS.md`)
